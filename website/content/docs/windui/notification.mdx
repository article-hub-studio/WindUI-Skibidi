## Notification System

Notifications are temporary messages displayed to inform users about events or status changes. They automatically disappear after a set duration.

The notification stack is responsive from mobile to desktop, keeps the newest item at the top, and limits the visible stack to five items by default. The default `Compact` appearance uses a dark capsule with a quiet border, optional icon, and a two-pixel timer. Long text stays clear of metadata, the close target, actions, and progress. Timed notifications pause while hovered.

## Capsule and Card Appearances

`Compact` is the default and matches a small system-notification capsule. `Card` increases the icon area and supports avatar plus timestamp metadata. `Glass` keeps the accented glass treatment for scripts that want it.

```luau
WindUI:Notify({
    Title = "Notification example",
    Content = "A compact capsule",
    Appearance = "Compact",
    Icon = "geist:bell",
})

WindUI:Notify({
    Title = "Anonim",
    Content = "A larger notification with metadata.",
    Appearance = "Card",
    Avatar = "rbxassetid://123456789",
    Timestamp = "May 23, 2025",
    Duration = false,
})
```

Set `Decorated = true` to add the style color wash and top accent while keeping the capsule layout.

## Action Buttons

Notifications support up to two compact actions in one mobile-friendly row.

```luau
WindUI:Notify({
    Title = "Connection lost",
    Content = "Reconnect now or dismiss this message.",
    Style = "Warning",
    Duration = false,
    Buttons = {
        {
            Title = "Reconnect",
            Callback = function(Notification)
                reconnect()
            end,
        },
        {
            Title = "Dismiss",
        },
    },
})
```

Buttons close the notification after their callback by default. Set `Close = false` or `CloseOnClick = false` to keep it open.

## Update and Timer Controls

The returned notification can be updated in place or paused manually without creating another card.

```luau
local Notice = WindUI:Notify({
    Title = "Downloading",
    Content = "Preparing files...",
    Duration = false,
})

Notice:Update({
    Title = "Download complete",
    Content = "All files are ready.",
    Duration = 3,
})

Notice:Pause()
Notice:Resume()
print(Notice:GetRemainingDuration())
```

`Update` currently supports `Title`, `Content`, and `Duration`. Set `Content = false` to hide the content row.

## Lifecycle Callbacks

```luau
WindUI:Notify({
    Title = "Saved",
    Style = "Success",
    OnOpen = function(Notification)
        print("Opened", Notification.Index)
    end,
    OnClose = function(Notification, Reason)
        print("Closed because", Reason)
    end,
})
```

Close reasons include `Timeout`, `Dismissed`, `Action`, `Overflow`, and `Manual`.

## Styles

Use `Style`, `Type`, or `Variant` to pick the notification preset. Aliases like `warn`, `successful`, and `green` are normalized automatically.

```luau
WindUI:Notify({
    Title = "Heads up",
    Content = "This is a notice-style notification.",
    Style = "Notice",
})

WindUI:Notify({
    Title = "Saved",
    Content = "Config saved successfully.",
    Style = "Success",
})

WindUI:Notify({
    Title = "Warning",
    Content = "Missing permission.",
    Style = "Warning",
    Duration = false, -- keep it open until closed
})
```
