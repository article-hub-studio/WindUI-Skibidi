## Notification System

Notifications are temporary messages displayed to inform users about events or status changes. They automatically disappear after a set duration.

The notification stack is responsive from mobile to desktop, keeps the newest item at the top, and limits the visible stack to five items by default. Long titles and content wrap inside the glass card without overlapping the icon, close button, actions, or progress bar. Timed notifications pause while hovered so users have enough time to read or select an action.

## Action Buttons

Notifications support up to two compact actions in one mobile-friendly row.

```luau
WindUI:Notify({
    Title = "Connection lost",
    Content = "Reconnect now or dismiss this message.",
    Style = "Warning",
    Duration = false,
    Buttons = {
        {
            Title = "Reconnect",
            Callback = function(Notification)
                reconnect()
            end,
        },
        {
            Title = "Dismiss",
        },
    },
})
```

Buttons close the notification after their callback by default. Set `Close = false` or `CloseOnClick = false` to keep it open.

## Update and Timer Controls

The returned notification can be updated in place or paused manually without creating another card.

```luau
local Notice = WindUI:Notify({
    Title = "Downloading",
    Content = "Preparing files...",
    Duration = false,
})

Notice:Update({
    Title = "Download complete",
    Content = "All files are ready.",
    Duration = 3,
})

Notice:Pause()
Notice:Resume()
print(Notice:GetRemainingDuration())
```

`Update` currently supports `Title`, `Content`, and `Duration`. Set `Content = false` to hide the content row.

## Lifecycle Callbacks

```luau
WindUI:Notify({
    Title = "Saved",
    Style = "Success",
    OnOpen = function(Notification)
        print("Opened", Notification.Index)
    end,
    OnClose = function(Notification, Reason)
        print("Closed because", Reason)
    end,
})
```

Close reasons include `Timeout`, `Dismissed`, `Action`, `Overflow`, and `Manual`.

## Styles

Use `Style`, `Type`, or `Variant` to pick the notification preset. Aliases like `warn`, `successful`, and `green` are normalized automatically.

```luau
WindUI:Notify({
    Title = "Heads up",
    Content = "This is a notice-style notification.",
    Style = "Notice",
})

WindUI:Notify({
    Title = "Saved",
    Content = "Config saved successfully.",
    Style = "Success",
})

WindUI:Notify({
    Title = "Warning",
    Content = "Missing permission.",
    Style = "Warning",
    Duration = false, -- keep it open until closed
})
```
