## Notification System

Notifications are temporary messages displayed to inform users about events or status changes. They automatically disappear after a set duration.

The notification stack is responsive from mobile to desktop, keeps the newest item at the top, and limits the visible stack to four items by default. The default `Compact` appearance uses a small toast surface, optional icon, and a short two-pixel timer. Long text stays clear of metadata, the close target, actions, and progress. Timed notifications pause while hovered.

WindUI uses Roblox's native [`UIShadow`](https://create.roblox.com/docs/reference/engine/classes/UIShadow) and individual `UICorner` radii when available. The dark surface overlay and old frame-shadow fallback are both disabled by default; unsupported clients simply render without a shadow unless `FallbackShadow = true` is requested.

## Notification Types

`Type` selects the renderer while `Style` selects its status color. `Normal` is the modern capsule/card renderer, `Window` is a Windows-inspired app card with an optional selector and actions, and `Originally` restores the compact classic WindUI presentation.

```luau
WindUI:Notify({
    Type = "Window",
    AppName = "WindUI",
    AppIcon = "lucide:app-window",
    Title = "Feature Launch Party",
    Content = "Studio S / Ballroom\n4:00 PM",
    Selection = {
        Value = "Going",
        Values = { "Going", "Maybe", "Not going" },
    },
    Buttons = {
        { Title = "RSVP", Close = false },
        { Title = "Dismiss" },
    },
    Duration = false,
})

WindUI:Notify({
    Type = "Originally",
    Title = "Classic WindUI",
    Content = "The original compact notification surface.",
})
```

## Capsule and Card Appearances

`Compact` is the default and matches a small system-notification capsule. `Card` increases the icon area and supports avatar plus timestamp metadata. `Glass` enables the Liquid Glass surface and keeps the accented treatment for scripts that want it.

```luau
WindUI:Notify({
    Title = "Notification example",
    Content = "A compact capsule",
    Appearance = "Compact",
    Icon = "geist:bell",
})

WindUI:Notify({
    Title = "Anonim",
    Content = "A larger notification with metadata.",
    Appearance = "Card",
    Avatar = "rbxassetid://123456789",
    Timestamp = "May 23, 2025",
    Duration = false,
})
```

Set `Decorated = true` to add the style color wash and a quiet accent rail while keeping the compact layout.

## Liquid Glass

Use `Appearance = "Glass"` or add `LiquidGlass = true` to any compact/card notification. The glass texture, translucent surface, border, and native shadow are separate layers, so the card stays readable without a solid black overlay.

```luau
WindUI:Notify({
    Title = "Saved successfully",
    Content = "Liquid Glass is active.",
    Appearance = "Compact",
    LiquidGlass = true,
    DarkOverlay = false,
    GlassTransparency = 0.78,
    Style = "Success",
})
```

## Native Shadow and Individual Corners

Use `Corners` to set each radius independently. `Shadow = false` disables the native shadow. Set `FallbackShadow = true` only when a non-native compatibility shadow is required.

```luau
WindUI:Notify({
    Title = "Saved",
    Content = "Your changes are ready.",
    Style = "Success",
    Decorated = true,
    Corners = {
        TopLeft = 18,
        TopRight = 18,
        BottomRight = 18,
        BottomLeft = 10,
    },
    ShadowBlur = 16,
    ShadowTransparency = 0.58,
})
```

## Action Buttons

Notifications support up to two compact actions in one mobile-friendly row.

```luau
WindUI:Notify({
    Title = "Connection lost",
    Content = "Reconnect now or dismiss this message.",
    Style = "Warning",
    Duration = false,
    Buttons = {
        {
            Title = "Reconnect",
            Callback = function(Notification)
                reconnect()
            end,
        },
        {
            Title = "Dismiss",
        },
    },
})
```

Buttons close the notification after their callback by default. Set `Close = false` or `CloseOnClick = false` to keep it open.

## Update and Timer Controls

The returned notification can be updated in place or paused manually without creating another card.

```luau
local Notice = WindUI:Notify({
    Title = "Downloading",
    Content = "Preparing files...",
    Duration = false,
})

Notice:Update({
    Title = "Download complete",
    Content = "All files are ready.",
    Duration = 3,
})

Notice:Pause()
Notice:Resume()
print(Notice:GetRemainingDuration())
```

`Update` currently supports `Title`, `Content`, and `Duration`. Set `Content = false` to hide the content row.

## Lifecycle Callbacks

```luau
WindUI:Notify({
    Title = "Saved",
    Style = "Success",
    OnOpen = function(Notification)
        print("Opened", Notification.Index)
    end,
    OnClose = function(Notification, Reason)
        print("Closed because", Reason)
    end,
})
```

Close reasons include `Timeout`, `Dismissed`, `Action`, `Overflow`, and `Manual`.

## Styles

Use `Style` or `Variant` to pick the status preset. Aliases like `warn`, `successful`, and `green` are normalized automatically. For backward compatibility, `Type = "Success"` still works when the value is not one of the three renderer names.

```luau
WindUI:Notify({
    Title = "Heads up",
    Content = "This is a notice-style notification.",
    Style = "Notice",
})

WindUI:Notify({
    Title = "Saved",
    Content = "Config saved successfully.",
    Style = "Success",
})

WindUI:Notify({
    Title = "Warning",
    Content = "Missing permission.",
    Style = "Warning",
    Duration = false, -- keep it open until closed
})
```
